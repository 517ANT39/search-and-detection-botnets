# Функциональные требования интеллектуальной системы мониторинга активности приложений

## Общие цели проекта:  
Система должна собирать данные работе приложений с множества машин сети организации, обеспечивая их хранение и доступность для последующего анализа с целью дообучения моделей агентов на хостах и составления отчетов по собранным данным.  

## Основные компоненты системы:
1.	Агенты мониторинга  
      *	Компоненты для автоматического мониторинга активности компьютеров сети предприятия
2.	Приложение сбору и обработке данных идущих от агентов 
      *	Обеспечивает сбор и хранение данных идущих от агентов.
3.	Модель для обработки собранных данных 
      *	Анализирует данные через опеределенные интервалы времени, и выгружает обработанные данные в витрины.
      * Обновляет базы угроз агентов мониторига
4.	Панель управления и отчетность  
   *	Интерфейс администрирования, инструменты визуализации результатов анализа и подготовки отчетов.

## Требования к функциональности системы:
### Сбор данных
1. Сбор данных в едином формате
2. Реализация механизма постоянного наблюдения за всеми активными приложениями и службами.
3. Сбор метрик и характеристик каждой программы (использование CPU, RAM, чтение-запись файлов, сетевое взаимодействие).
4. Запись журнала событий активности приложений с указанием точного времени, исполняемого файла, типа события (создание нового процесса, доступ к файлам, сетевые соединения и т.п.)
5. Идентификация аномальных типов активности путем анализа собранных метрик.
6. Выявление признаков ботов и иных проявлений несанкционированной активности.
7. Обработка потоков больших объемов данных: Поддержка потокового ввода данных с минимизацией задержек и потерь пакетов.

### Приложение сбору и обработке данных идущих от агентов
1. Централизованное хранилище: Обеспечение единого места хранения данных для всех компонентов системы.
2. Автоматизация архивации: Регулярная архивация старых данных.
3. Безопасность доступа: Предоставление механизмов аутентификации и авторизации для предотвращения несанкционированного доступа к данным.

### Анализ и обработка данных
1.	Алгоритмы предобработки: Механизмы нормализации, фильтрации, обогащения данных перед отправкой на анализ.
2.	Алгоритмы анализа данных: Модели детектирования аномального поведения пользователей и служб.
3. Отправка новых "дообученных" моделей агентам мониторинга

### Панель управления и отчеты
1. Интерфейсы визуализации: Дашборды с возможностью интерактивного просмотра текущих событий и тенденций.
2. Отчеты и статистика: Автоматизированная подготовка регулярных отчетов и статистической отчетности.
3. Импорт/экспорт данных: Средства экспорта данных для сторонних инструментов анализа и импортируемых сведений для расширенного анализа вне системы.
