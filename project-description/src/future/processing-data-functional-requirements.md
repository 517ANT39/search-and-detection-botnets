# Функциональные требования к обработке и хранению данных поступающих с машин предприятия

## Общие цели проекта:  
Система должна собирать данные с множества машин сети организации, сохранять их в Lakehouse, обеспечивать их хранение и доступность для последующего анализа с целью обнаружения аномалий, подозрительных действий пользователей, сетевых атак, включая выявление ботнетов и других вредоносных активностей.  

## Основные компоненты системы:
1.	Сборщики данных  
      *	Компоненты для автоматического мониторинга активности компьютеров сети предприятия
2.	Lakehouse  
      *	Хранилище необработанных сырых данных большого объема, обеспечивающее долгосрочное сохранение всех видов собранных данных.
3.	Средства обработки и аналитики  
      *	Инструменты и алгоритмы для предварительной обработки, очистки, агрегирования и анализа данных.
4.	Панель управления и отчетность  
   *	Интерфейс администрирования, инструменты визуализации результатов анализа и подготовки отчетов.

## Требования к функциональности системы:
### Сбор данных
1.	Сбор данных в едином формате
2.	Реализация агентов мониторинга: Легко устанавливаемые и настраиваемые модули мониторинга, работающие локально на машинах и собирающие данные для передачи в систему.
3.	Защита передаваемых данных: Шифрование каналов связи между агентами и сервером сбора данных.
4.	Обработка потоков больших объемов данных: Поддержка потокового ввода данных с минимизацией задержек и потерь пакетов.

### Lakehouse
1. Централизованное хранилище: Обеспечение единого места хранения данных для всех компонентов системы.
2. Автоматизация архивации: Регулярное создание резервных копий и архивация старых данных.
3. Безопасность доступа: Предоставление механизмов аутентификации и авторизации для предотвращения несанкционированного доступа к данным.
4. Метаданные и каталогизация: Хранение метаданных о структуре данных и их источниках для упрощения дальнейшей обработки.

### Анализ и обработка данных
1.	Алгоритмы предобработки: Механизмы нормализации, фильтрации, обогащения данных перед отправкой на анализ.
2.	Алгоритмы анализа данных: Модели детектирования аномального поведения пользователей и служб.
3. Отправка новых "дообученных" моделей агентам мониторинга

### Панель управления и отчеты
1. Интерфейсы визуализации: Дашборды с возможностью интерактивного просмотра текущих событий и тенденций.
2. Отчеты и статистика: Автоматизированная подготовка регулярных отчетов и статистической отчетности.
3. Импорт/экспорт данных: Средства экспорта данных для сторонних инструментов анализа и импортируемых сведений для расширенного анализа вне системы.
